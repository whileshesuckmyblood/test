Идея проекта - разработка и развертывание ТГ-бота с готовой прод. инфраструктурой.
Цель - продемонстрировать навыки работы со стеком.

Стек:
- **python 3.11 + aiogram 3.13.0** - логика бота, работает через вебхуки.
- **nginx** (reverse proxy + lets encrypt TLS) - обрабатывает HTTPS запросы и перенаправляет их в контейнер бота.
- **Docker** - контейнеризация приложения, гарантирует одинаковую стабильную работу на разных устройствах.
- **docker compose** - оркестратор контейнеров, в проекте используется два контейнера nginx и tg-bot, управляет запуском, сетями и зависимостями.
- **GitHub Actions** (CI/CD) - используется автоматическая сборка, тестирование и деплой приложения на VPS через SSH, при каждом коммите нашего кода в репозиторий.
- **VPS (Linux)** - Ubuntu24.04.
- **Мониторинг** ресурсов Prometheus(сбор и хранение метрик), Node Exporter(сбор системных метрик на хосте) + Grafana(визуализация, алерты)

Как работает: 
Пользователь пишет боту, API телеграма направляет сообщение HTTPS запрос на домен, nginx принимает запрос на 443 порту, производит расшифровку через TLS(lets encrypt), и после этого перенаправляет запрос по внутренней сети Докера к контейнеру с ботом на 8080 порту. Бот обрабатывает данные и возвращает ответ обратно через прокси. Контейнер бота не открывает порты во внешний мир, а работает внутри своей сети.
telegram -> HTTPS(443) -> nginx (reverse proxy + TLS) --(внутрення сеть докера)--> контейнер бота (порт 8080) -> ответ пользователю

CI/CD:
CI(непрерывная интеграция) - при пуше кода в ветку main, запускается workflow, Actions проверяет репозиторий, устанавливает python 3.11 и зависимости, запускает тесты через pytest, собирается образ бота с тегом коммита SHA, образ пушится в реестр контейнеров гитхаба(GHCR) для последующего деплоя.
CD(непрерывное развертывание) - после того, как успешно прошел этап интеграции - начинается этот, деплой кода на VPS через SSH, происходит логин в GHCR и пулл последнего docker-образа. Затем контейнер пересоздается с новым образом.

Благодаря этим двум этапам, каждое изменение кода автоматически обновляет бота на сервере без ручного вмешательства, что очень удобно!

Безопасность:
- Секреты хранятся в GitHub Secrets.
- HTTPS обеспечивается с помощью nginx и let's encrypt.
- Никаких конфиденциальных данных в репозитории.

Мониторинг:
Prometheus собирает метрики системы, Node Exporter выдает системные метрики хоста, Grafana используется для визуализации и алертов на основе данных от Prometheus.
